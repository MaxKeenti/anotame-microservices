# QUICK TASK: Fix pricelists 404 error when creating orders

## Root Cause

`price-list-step.svelte` constructs URLs as `${API_CATALOG}/api/pricelists` which resolves to `/api/catalog/api/pricelists`. After proxy stripping, this forwards to `http://localhost:8082/api/pricelists` — but the catalog-service backend exposes `GET /pricelists` (no `/api` prefix).

Every other call in the codebase correctly uses `${API_CATALOG}/pricelists`.

## Fix

Remove the extra `/api/` from both URL constructions in `price-list-step.svelte`:
- Line 31: `${API_CATALOG}/api/pricelists` → `${API_CATALOG}/pricelists`
- Line 63: `${API_CATALOG}/api/pricelists/${priceListId}` → `${API_CATALOG}/pricelists/${priceListId}`
