<objective>
Migrate the service price overrides table on the price list detail page (`/dashboard/catalog/pricelists/[id]`) from custom Table components to DataTableWrapper, maintaining editable inline inputs while standardizing table behavior across the application.

**Context:** DataTableWrapper unifies filtering, sorting, and pagination. The overrides table currently uses custom shadcn Table components. While it has inline editable Input fields, DataTableWrapper can be extended to support this pattern consistently.

**Why:** Consolidates table UI/UX patterns (STATE.md: "DataTableWrapper filter deduplication is a priority bug fix"). Sets precedent for editable tables using DataTableWrapper.
</objective>

<background>

**Current Implementation:** `/dashboard/catalog/pricelists/[id]`
- Uses shadcn `Table.Root`, `Table.Header`, `Table.Row` components (manual rendering)
- Shows 3 columns: Service name, Base Price (read-only), Override Price (editable Input)
- Editable override inputs bound to state: `overrides: Record<string, string>`
- No filtering/sorting (all services shown in memory)
- Bulk adjustment buttons (+5, +10, +15, +20 and corresponding negatives)
- Custom hover states and styling

**DataTableWrapper Current Support:**
- TanStack table-core with sorting, filtering, pagination
- `actionCell` snippet for custom rendering in 'actions' column only
- All other cells render via `cell.getValue()` (text/string values)
- **Gap:** No support for custom cell rendering outside 'actions' column, no support for interactive elements (Inputs, buttons) within table cells

**Why Extension Needed:**
Price override table requires editable Input elements that bind to parent state. DataTableWrapper's current `cell.getValue() as string` pattern doesn't support this. Solution: Extend DataTableWrapper with `cellRenders` prop (map of column ID → snippet function) similar to existing `actionCell`.

</background>
