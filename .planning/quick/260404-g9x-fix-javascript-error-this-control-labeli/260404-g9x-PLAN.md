  <action>
The issue is that FormPrimitive.Label is being rendered with a snippet before the Control context is guaranteed to exist. The solution is to use `asChild` mode when direct context binding isn't available, or wrap the rendering to delay until Control is initialized.

Update form-label.svelte:
1. Change the FormPrimitive.Label pattern to use asChild=true, which skips formsnap's internal label element and renders only the child snippet content
2. The Label component from shadcn will still receive proper aria-labelledby through the snippet props when available
3. Keep the fallback Label component that doesn't require formsnap context

Specific changes:
- Line 14: Change `<FormPrimitive.Label>` to `<FormPrimitive.Label asChild>`
- This tells formsnap to not render its own label element, just execute the snippet
- The `{...props}` from the snippet will still contain proper label attributes when Control context exists
- When Control context doesn't exist, props will be minimal but won't cause errors

This is a safe pattern used in shadcn-svelte templates and maintains full functionality while preventing the undefined context error.
