<task type="auto">
  <name>Task 2: Create dev.sh — full local dev launcher</name>
  <files>dev.sh</files>
  <action>
Create an executable dev.sh at the repo root. The script must:

1. Accept an optional `stop` subcommand: `./dev.sh stop` kills all background PIDs tracked in
   /tmp/anotame-dev.pids and exits.

2. Default (no args) — start sequence:
   a. `docker compose up -d` — start the 4 DBs (already defined in docker-compose.yml at root).
      Poll until all 4 DB containers are healthy (`docker inspect --format='{{.State.Health.Status}}'`).
   b. Launch each Quarkus service in the background using the Maven wrapper with the dev profile.
      Working directory for mvnw MUST be `anotame-api/backend/` (the Maven multi-module root).
      Commands (run in background, output redirected to per-service log files):
        - identity-service  → `./mvnw quarkus:dev -pl identity-service  -Dquarkus.http.port=8081`
        - catalog-service   → `./mvnw quarkus:dev -pl catalog-service   -Dquarkus.http.port=8082`
        - sales-service     → `./mvnw quarkus:dev -pl sales-service     -Dquarkus.http.port=8083`
        - operations-service→ `./mvnw quarkus:dev -pl operations-service -Dquarkus.http.port=8084`
      Log files: `/tmp/anotame-{service}.log` (e.g. `/tmp/anotame-identity-service.log`).
   c. Launch SvelteKit frontend in the background:
        Working directory: `anotame-web/`
        Command: `bun run dev`
        Log file: `/tmp/anotame-frontend.log`
   d. Write all background PIDs to `/tmp/anotame-dev.pids` (one per line), so `stop` can kill them.
   e. Print a summary table to stdout showing each service name, port/URL, and log file path.
   f. Tail all log files in the foreground so the terminal shows combined output. On Ctrl+C,
      run the stop routine (kill all PIDs, `docker compose stop`).

3. Script preamble:
   - `#!/usr/bin/env bash`
   - `set -euo pipefail`
   - Define REPO_ROOT via `$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)` so the script works
     from any cwd.
   - Change to REPO_ROOT early.

4. Make it executable: `chmod +x dev.sh`.

Ports (from application.properties, confirmed):
  - identity-service:   http://localhost:8081
  - catalog-service:    http://localhost:8082
  - sales-service:      http://localhost:8083
  - operations-service: http://localhost:8084
  - frontend (SvelteKit): http://localhost:5173

DB healthcheck: wait up to 60 s, checking every 3 s. If any DB is not healthy after timeout,
print an error and exit 1 (do not start services against an unhealthy DB).

Do NOT use tmux — plain background processes + tail keeps it dependency-free.
  </action>

  </verify>
