<objective>
Fix dashboard role filtering so employees cannot access KPI or other admin-only content.

Purpose: Security vulnerability — non-admin users can access admin pages like KPI by navigating directly to /dashboard/admin/* routes. The menu filters client-side, but client-side filtering is bypassable.

Output: Server-side authorization checks on all admin routes + client-side guards on admin pages.
</objective>

<execution_context>
The issue is that authorization is currently client-side only. The `useAuthGuard(true)` guard exists in the codebase but is not applied to admin pages. Admin routes have no server-side protection and can be accessed by any authenticated user.

Key findings:
- Auth service stores role in localStorage (client-side persisted state)
- `useAuthGuard(true)` exists and can enforce ADMIN-only access
- Admin pages (KPI, Users, Schedule, Settings) don't use the guard
- The main dashboard filters menu items client-side but doesn't prevent direct URL access
</execution_context>
