# Phase 2 Plan 03: Add Security Annotations to Controllers

**Phase:** 2
**Plan:** 03
**Goal:** Enforce authentication on all unguarded endpoints in operations-service and identity-service, and restrict mutation endpoints in `UserController` to the ADMIN role.
**Req IDs:** SEC-03, SEC-04

---

## Overview

Four controllers have zero security annotations and accept unauthenticated requests: `OperationsController`, `ScheduleController`, `EstablishmentController` (operations-service), and `UserController` (identity-service). This plan adds `@io.quarkus.security.Authenticated` at class level to each, following the existing pattern from `OrdersResource.java` in sales-service. For `UserController`, `@jakarta.annotation.security.RolesAllowed("ADMIN")` is additionally placed on the `POST`, `PUT`, and `DELETE` methods. `GET /schedule/check` is exempted with `@jakarta.annotation.security.PermitAll` because it may be consumed by the guest-facing booking widget without a JWT.

`AuthController` already has method-level `@io.quarkus.security.Authenticated` on `/auth/me` and `/auth/change-credentials` — and no class-level annotation. This is intentional: `POST /auth/login`, `POST /auth/register`, `POST /auth/logout` must remain public. Do NOT add class-level `@Authenticated` to `AuthController`.

### Task 2: Secure `ScheduleController` with `@PermitAll` on `GET /schedule/check`

**File**: `anotame-api/backend/operations-service/src/main/java/com/anotame/operations/infrastructure/web/controller/ScheduleController.java`

**Action**: Add `@io.quarkus.security.Authenticated` at class level AND add `@jakarta.annotation.security.PermitAll` on the `checkAvailability` method (the `GET /schedule/check` endpoint).

The current class declaration block is:

```java
@Path("/schedule")
@Produces(MediaType.APPLICATION_JSON)
@Consumes(MediaType.APPLICATION_JSON)
@RequiredArgsConstructor
public class ScheduleController {
```

Change to:

```java
@Path("/schedule")
@Produces(MediaType.APPLICATION_JSON)
@Consumes(MediaType.APPLICATION_JSON)
@RequiredArgsConstructor
@io.quarkus.security.Authenticated
public class ScheduleController {
```

The current `checkAvailability` method signature is:

```java
    @GET
    @Path("/check")
    public Map<String, Boolean> checkAvailability(@QueryParam("date") String dateStr) {
```

Change to:

```java
    @GET
    @Path("/check")
    @jakarta.annotation.security.PermitAll
    public Map<String, Boolean> checkAvailability(@QueryParam("date") String dateStr) {
```

No other method receives additional annotations. `GET /schedule/config`, `PUT /schedule/config`, `GET /schedule/holidays`, `POST /schedule/holidays`, `DELETE /schedule/holidays/{id}` all inherit class-level `@Authenticated`.

**Verify**: Build:

Must exit with code 0. Confirm both annotations present:
```bash

  anotame-api/backend/operations-service/src/main/java/com/anotame/operations/infrastructure/web/controller/ScheduleController.java
```
Must return at least 2 matches: class-level `Authenticated` and method-level `PermitAll`.

### Task 3: Secure `UserController` with class-level auth and ADMIN role on mutations

**File**: `anotame-api/backend/identity-service/src/main/java/com/anotame/identity/infrastructure/web/controller/UserController.java`

**Action**: Add `@io.quarkus.security.Authenticated` at class level AND add `@jakarta.annotation.security.RolesAllowed("ADMIN")` on each of the `POST`, `PUT`, and `DELETE` methods.

The current class declaration block is:

```java
@Path("/users")
@Produces(MediaType.APPLICATION_JSON)
@Consumes(MediaType.APPLICATION_JSON)
@RequiredArgsConstructor
public class UserController {
```

Change to:

```java
@Path("/users")
@Produces(MediaType.APPLICATION_JSON)
@Consumes(MediaType.APPLICATION_JSON)
@RequiredArgsConstructor
@io.quarkus.security.Authenticated
public class UserController {
```

The current method signatures that need `@RolesAllowed`:

```java
    @POST
    public UserResponse createUser(com.anotame.identity.application.dto.CreateUserRequest request) {
```

```java
    @PUT
    @Path("/{id}")
    public UserResponse updateUser(@PathParam("id") UUID id, UpdateUserRequest request) {
```

```java
    @DELETE
    @Path("/{id}")
    public void deleteUser(@PathParam("id") UUID id) {
```

Change each by adding `@jakarta.annotation.security.RolesAllowed("ADMIN")` on the line immediately before the HTTP verb annotation:

```java
    @jakarta.annotation.security.RolesAllowed("ADMIN")
    @POST
    public UserResponse createUser(com.anotame.identity.application.dto.CreateUserRequest request) {
```

```java
    @jakarta.annotation.security.RolesAllowed("ADMIN")
    @PUT
    @Path("/{id}")
    public UserResponse updateUser(@PathParam("id") UUID id, UpdateUserRequest request) {
```

```java
    @jakarta.annotation.security.RolesAllowed("ADMIN")
    @DELETE
    @Path("/{id}")
    public void deleteUser(@PathParam("id") UUID id) {
```

The `GET` methods (`getAllUsers` and `getUserById`) do NOT receive `@RolesAllowed` — any authenticated user can read the user list (HTTP-level policy `authenticated` on `/users/*` already handles unauthenticated access at the perimeter; the CDI `@Authenticated` adds defense-in-depth).

Do NOT modify `AuthController.java`. It already has method-level `@io.quarkus.security.Authenticated` on `/auth/me` and `/auth/change-credentials`, and public endpoints (`/auth/login`, `/auth/register`, `/auth/logout`) have no annotation — which is correct.

**Verify**: Build identity-service:
```bash
cd anotame-api/backend && ./mvnw compile -pl identity-service -am -q
```
Must exit with code 0. Confirm annotations:
```bash
grep -n "Authenticated\|RolesAllowed" \
  anotame-api/backend/identity-service/src/main/java/com/anotame/identity/infrastructure/web/controller/UserController.java
```
Must return 4 matches: 1 class-level `Authenticated` and 3 method-level `RolesAllowed`.

Confirm `AuthController` was not modified:
```bash
grep -n "class.*Authenticated\|class AuthController" \
  anotame-api/backend/identity-service/src/main/java/com/anotame/identity/infrastructure/web/controller/AuthController.java
```
Must NOT show class-level `@Authenticated` on `AuthController` itself.

### Task 4: Integration test via `docker compose up --build` and curl

**Action**: Start all services with a full build:

```bash
docker compose up --build -d
```

Wait for all services to be healthy (check with `docker compose ps`). Then run the following curl tests. Replace `<EMPLOYEE_JWT>` and `<ADMIN_JWT>` with tokens obtained from `POST /auth/login` with employee and admin credentials respectively.

**Test 1 — Unauthenticated request to operations returns 401:**
```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:8084/operations/work-orders
```
Expected: `401`

**Test 2 — Authenticated employee can read work orders:**
```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:8084/operations/work-orders \
  -H "Cookie: jwt=<EMPLOYEE_JWT>"
```
Expected: `200`

**Test 3 — Unauthenticated availability check is permitted:**
```bash
curl -s -o /dev/null -w "%{http_code}" \
  "http://localhost:8084/schedule/check?date=2026-06-15T10:00:00"
```
Expected: `200` (no auth required)

**Test 4 — Employee cannot delete a user (ADMIN-only):**
```bash
curl -s -o /dev/null -w "%{http_code}" -X DELETE \
  http://localhost:8081/users/<any-uuid> \
  -H "Cookie: jwt=<EMPLOYEE_JWT>"
```
Expected: `403`

**Test 5 — Admin can delete a user:**
```bash
curl -s -o /dev/null -w "%{http_code}" -X DELETE \
  http://localhost:8081/users/<any-uuid> \
  -H "Cookie: jwt=<ADMIN_JWT>"
```
Expected: `200` or `404` (not `401` or `403`)

**Test 6 — Login still works (AuthController not broken):**
```bash
curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:8081/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"password"}'
```
Expected: `200`

**Verify**: All 6 tests return the expected HTTP status codes.
