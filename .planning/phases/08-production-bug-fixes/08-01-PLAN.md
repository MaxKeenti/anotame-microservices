  <action>
The root cause is the `$effect` at lines 47â€“50:

```svelte
$effect(() => {
  void globalFilter;
  pagination = { pageIndex: 0, pageSize: pagination.pageSize };
});
```

This effect reads `globalFilter` (tracked) and writes to `pagination`. But `pagination` is also read inside the `$derived(table)` block (lines 52â€“96). In Svelte 5, when a `$derived` is re-evaluated, any `$state` writes inside effects that also depend on derived values can form a cycle.

The fix: wrap the `pagination` write in `untrack` so the assignment is not tracked as a reactive dependency, breaking the cycle:

```svelte
import { untrack } from 'svelte';

$effect(() => {
  void globalFilter;
  untrack(() => {
    pagination = { pageIndex: 0, pageSize: pagination.pageSize };
  });
});
```

Steps:
1. Add `untrack` to the existing `svelte` import at the top of the `<script>` block. The current import is:
   ```ts
   import {
     createTable,
     getCoreRowModel,
     ...
   } from '@tanstack/table-core';
   ```
   `untrack` is imported from `'svelte'`, not from `@tanstack/table-core`. Add a new import line:
   ```ts
   import { untrack } from 'svelte';
   ```

2. Replace the existing `$effect` block (lines 47â€“50) with the `untrack`-wrapped version shown above.

Do NOT change any other part of DataTableWrapper â€” the public API (columns, data, loading, emptyMessage, filterPlaceholder, actionCell snippet) must remain unchanged (per D-04).
