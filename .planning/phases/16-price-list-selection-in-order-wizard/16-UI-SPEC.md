|------|-------|-------|

## Interaction States

### Loading state (price lists fetching)

```
Trigger: onMount fetch of GET /api/catalog/pricelists
Display: AdaptiveSelect trigger shows "Cargando listas..." with Loader2 icon (w-4 h-4 animate-spin)
Selector: disabled during fetch
Duration: Until promise resolves or rejects
```

### Empty state (no active price lists returned)

```
Trigger: fetch succeeds but returns [] or all lists have active: false
Display: Inline info box below selector
  Icon: AlertTriangle (w-5 h-5, text-warning)
  Heading: "No hay listas de precios disponibles"
  Body: "El catálogo no tiene listas activas. Continúa sin lista o pide a un administrador que active una."
  Button: none — step remains non-blocking, staff continues without list
Selector: Shows only "Sin lista de precios" option
```

### Error state (API fetch fails)

```
Trigger: apiService throws on GET /api/catalog/pricelists
Display: toast.error("No se pudo cargar las listas de precios") — svelte-sonner
  + Inline fallback message in step body:
    "Ocurrió un error al cargar las listas. Puedes continuar sin una lista de precios."
Selector: Shows only "Sin lista de precios" option (step remains non-blocking)
Recovery: Retry button — "Reintentar" (outline, small) triggers re-fetch
```

### Selection loading state (items hydration)

```
Trigger: Staff selects a price list from dropdown (not "Sin lista de precios")
Display: Loader2 spinner replaces confirmation card during GET /api/catalog/pricelists/{id}
Duration: Until items are stored in draft or error occurs
```

### Selection error state (item hydration fails)

```
Trigger: GET /api/catalog/pricelists/{id} throws after selection
Display: toast.error("Error al cargar la lista de precios. Intenta de nuevo.")
Result: Selection is cleared — draft.priceListId reverts to undefined
```

```
toast.success("{priceListName}", { description: "Lista de precios aplicada" })
```
i18n key for description: `price_list_applied_description`

### No destructive actions in this phase

The price list step has no deletions, no confirmations via `adaptiveConfirm()`. The only user-initiated "undo" is selecting "Sin lista de precios" from the dropdown, which silently clears the draft field. No confirmation dialog is needed.

## Accessibility Contract

Required per AI_RULES.md ("A11y Strictness"):

- The `AdaptiveSelect` trigger must have an associated `<label>` via `for=` / `id=` linkage.
- The price list confirmation card (when rendered) is `role="status"` — it updates in place on selection.
- The loading spinner (`Loader2`) requires `aria-label="Cargando"` on its wrapper.
- The read-only edit display card uses `role="region"` with `aria-label="Lista de precios seleccionada"`.
- The active price list badge in `ItemsStep` uses `aria-label="Lista de precios activa: {name}"`.
- All interactive buttons meet the 44px minimum touch target (enforced via `h-12` or `h-14` Tailwind classes).
