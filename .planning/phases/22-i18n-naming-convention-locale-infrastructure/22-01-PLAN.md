<task id="3" type="execute">
<title>Create i18n key naming convention lint script</title>
<read_first>
- .planning/research/FEATURES.md (§5 Message-Key Naming Convention)
- anotame-web/package.json
</read_first>
<action>
1. Create `anotame-web/scripts/lint-i18n-keys.js`:
   ```javascript
   #!/usr/bin/env node
   /**
    * Lint i18n message keys against the naming convention.
    * Convention: domain.component.purpose (2-3 camelCase segments)
    * Regex: /^[a-z][a-zA-Z]*(\.[a-z][a-zA-Z]*){1,2}$/
    */
   import { readFileSync, readdirSync } from 'fs';
   import { join } from 'path';

   const KEY_REGEX = /^[a-z][a-zA-Z]*(\.[a-z][a-zA-Z]*){1,2}$/;
   const MESSAGES_DIR = join(import.meta.dirname, '..', 'messages');

   let errors = 0;
   const files = readdirSync(MESSAGES_DIR).filter(f => f.endsWith('.json'));

   for (const file of files) {
     const content = JSON.parse(readFileSync(join(MESSAGES_DIR, file), 'utf-8'));
     for (const key of Object.keys(content)) {
       if (!KEY_REGEX.test(key)) {
         console.error(`❌ ${file}: "${key}" does not match naming convention`);
         errors++;
       }
     }
   }

   if (errors > 0) {
     console.error(`\n${errors} key(s) violate naming convention.`);
     console.error('Convention: domain.component.purpose (2-3 camelCase segments)');
     console.error('Regex: /^[a-z][a-zA-Z]*(\\.[a-z][a-zA-Z]*){1,2}$/');
     process.exit(1);
   } else {
     console.log(`✓ ${files.length} file(s), all keys valid.`);
   }
   ```

2. Add script to `anotame-web/package.json` scripts:
   ```json
   "lint:i18n": "node scripts/lint-i18n-keys.js"
   ```

3. Run `bun run lint:i18n` to verify it passes with the seed messages.
</action>
<acceptance_criteria>
- anotame-web/scripts/lint-i18n-keys.js exists
- anotame-web/package.json contains "lint:i18n" script
- Running `bun run lint:i18n` in anotame-web exits 0 with "all keys valid" message
- A key like "BAD_KEY" or "too.many.segments.here" would cause the script to exit 1
</acceptance_criteria>
</task>
