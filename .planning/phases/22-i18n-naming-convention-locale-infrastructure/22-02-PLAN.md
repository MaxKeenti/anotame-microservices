# Plan 22-02: Identity Service Locale Column + PATCH Endpoint

<objective>
Add a `locale` column to `tca_user`, surface it in UserResponse (so login returns the user's locale), and expose a PATCH endpoint for updating locale preference — completing the backend half of per-user locale persistence.
</objective>

<tasks>

<task id="1" type="execute">
<title>Add V2 Flyway migration for locale column</title>
<read_first>
- anotame-api/backend/identity-service/src/main/resources/db/migration/V1__baseline.sql
</read_first>
<action>
Create `anotame-api/backend/identity-service/src/main/resources/db/migration/V2__add_user_locale.sql`:
```sql
-- Add locale preference column to tca_user
-- Default 'es' (Spanish) — Paraglide sourceLanguageTag
ALTER TABLE tca_user ADD COLUMN locale VARCHAR(10) DEFAULT 'es' NOT NULL;

-- Set all existing users to 'es' (explicit, even though DEFAULT handles it)
COMMENT ON COLUMN tca_user.locale IS 'User locale preference for UI language (BCP-47 tag: es, en)';
```

Use `es` (not `es-MX`) to match Paraglide's `sourceLanguageTag`. The frontend maps `es` → display label "Español (México)".
</action>
<acceptance_criteria>
- File V2__add_user_locale.sql exists in db/migration/
- Contains `ALTER TABLE tca_user ADD COLUMN locale VARCHAR(10) DEFAULT 'es' NOT NULL`
- Does NOT drop or modify any existing columns
</acceptance_criteria>
</task>

<read_first>
- anotame-api/backend/identity-service/src/main/java/com/anotame/identity/application/service/UserService.java
</read_first>
<action>
Add a transactional method to `UserService.java`:
```java
@Transactional
public void updateLocale(UUID userId, String locale) {
    var user = userRepository.findById(userId);
    if (user == null) {
        throw new ResourceNotFoundException("User");
    }
    user.setLocale(locale);
}
```

Import `com.anotame.identity.domain.exception.ResourceNotFoundException` if not already imported.
</action>

<task id="7" type="execute">
<title>Add PATCH locale endpoint to UserController</title>
<read_first>
- anotame-api/backend/identity-service/src/main/java/com/anotame/identity/infrastructure/web/controller/UserController.java
</read_first>
<action>
Add a new PATCH endpoint to `UserController.java`:
```java
@PATCH
@Path("/{id}/locale")
public Response updateLocale(@PathParam("id") UUID id,
                             @jakarta.validation.Valid UpdateLocaleRequest request) {
    userService.updateLocale(id, request.getLocale());
    return Response.noContent().build();
}
```

Add the import for `UpdateLocaleRequest`:
```java
import com.anotame.identity.application.dto.UpdateLocaleRequest;
```

The endpoint is `@Authenticated` (inherited from class-level annotation) but NOT `@RolesAllowed("ADMIN")` — any authenticated user can change their own locale.
</action>
<acceptance_criteria>
- UserController.java contains `@PATCH` and `@Path("/{id}/locale")`
- Method signature: `public Response updateLocale(@PathParam("id") UUID id, @jakarta.validation.Valid UpdateLocaleRequest request)`
- Returns `Response.noContent().build()` (204)
- No `@RolesAllowed("ADMIN")` on this endpoint
- Import for UpdateLocaleRequest present
</acceptance_criteria>
</task>
