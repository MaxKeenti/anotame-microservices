<objective>
Rewrite the V1__baseline.sql for identity-service and catalog-service so each contains
only the tables that service owns — no pg_dump noise, no phantom tables from the shared DB.
Also clean up application.properties for both services to remove the shared-DB Flyway
workarounds (baseline-on-migrate, baseline-version, custom table name).

Background: Every service currently has an identical V1 that is a full pg_dump of the
shared anotame database — all 17 tables appear in every V1. With independent databases
per service, each V1 must create only that service's tables. identity-service and
catalog-service are the simplest cases: neither has incremental migrations (no V2+),
and neither has cross-service DB-level FKs to drop.

Out of scope for this plan: updating `quarkus.datasource.jdbc.url` to point at the
per-service database. That is Phase 19 work (CFG-01 through CFG-03). This plan only
removes the Flyway workaround flags that were needed when all services shared one DB.
</objective>

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto WITH SCHEMA public;
CREATE EXTENSION IF NOT EXISTS citext WITH SCHEMA public;

    description VARCHAR(255),

    deleted_at  TIMESTAMPTZ,
    is_deleted  BOOLEAN DEFAULT FALSE NOT NULL
);

CREATE TABLE tca_user (

    username        VARCHAR(255) NOT NULL UNIQUE,
    email           VARCHAR(255) UNIQUE,
    password_hash   VARCHAR(255) NOT NULL,
    first_name      VARCHAR(255),
    last_name       VARCHAR(255),
    last_login_at   TIMESTAMPTZ,
    is_active       BOOLEAN DEFAULT TRUE NOT NULL,
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ,
    is_deleted      BOOLEAN DEFAULT FALSE NOT NULL
);
```
