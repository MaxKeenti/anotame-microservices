# Phase 4 Plan 03: Gate SQL Logging to %dev Profile (all 4 services)

**Phase:** 4
**Plan:** 03
**Goal:** SQL query logging is absent in production (Railway) logs and present only in local `%dev` profile across all 4 services.
**Req IDs:** QUAL-03
**Depends on:** none (config-only change, fully independent)

---

## Overview

All 4 services set `quarkus.hibernate-orm.log.sql=true` with no profile prefix. This means every SQL query Hibernate executes is emitted in production Railway logs, creating noise and potentially exposing schema information.

Additionally, all 4 services set `quarkus.hibernate-orm.sql-formatting=true`. This is a non-standard property name — the correct Quarkus 3.x key is `quarkus.hibernate-orm.log.format-sql`. The current key is likely silently ignored by Quarkus (Hibernate receives no formatting instruction), so SQL may appear as a single long line even in dev. The fix corrects both: the `false` default and the property name.

The fix is identical for all 4 services: replace the ungated `true` values with `false` defaults and add `%dev.` prefixed overrides.

---

## Tasks

### Task 1: Update `identity-service` application.properties

**File**: `anotame-api/backend/identity-service/src/main/resources/application.properties`

**Action**: Edit. In the `# JPA / Hibernate Configuration` section, replace:

```properties
quarkus.hibernate-orm.log.sql=true
quarkus.hibernate-orm.sql-formatting=true
```

With:

```properties
quarkus.hibernate-orm.log.sql=false
quarkus.hibernate-orm.log.format-sql=false
%dev.quarkus.hibernate-orm.log.sql=true
%dev.quarkus.hibernate-orm.log.format-sql=true
```

Leave all other properties (`database.generation`, `mapping.format.global`, CORS, JWT, etc.) unchanged.

**Verify**: Confirm the two replaced lines are gone and the four replacement lines are present:
```bash
grep "hibernate-orm.log.sql\|hibernate-orm.log.format-sql\|hibernate-orm.sql-formatting" \
  anotame-api/backend/identity-service/src/main/resources/application.properties
```
Expected output (4 lines, no `sql-formatting`):
```
quarkus.hibernate-orm.log.sql=false
quarkus.hibernate-orm.log.format-sql=false
%dev.quarkus.hibernate-orm.log.sql=true
%dev.quarkus.hibernate-orm.log.format-sql=true
```
