# Phase 4 Plan 02: Typed Domain Exceptions in identity-service

**Phase:** 4
**Plan:** 02
**Goal:** Replace all bare `RuntimeException` throws in `AuthService` and `UserService` with typed domain exceptions that map to correct HTTP status codes — invalid-credentials login returns 401, duplicate-username registration returns 409.
**Req IDs:** QUAL-02
**Depends on:** 04-01 (requires `DomainException` base class and `GlobalExceptionHandler` created in Plan 04-01)

---

## Overview

`AuthService` has 7 `throw new RuntimeException(...)` sites and `UserService` has 4. Because there is no `GlobalExceptionHandler` and no typed exceptions, any of these throws currently produces HTTP 500 with either an HTML error page or a default Quarkus JSON error — not the semantically correct 401 or 409.

Plan 04-01 adds the `GlobalExceptionHandler` and `DomainException` base class to identity-service. This plan adds three concrete exception subclasses and replaces every `RuntimeException` throw site in the two service classes. No controller changes are needed: the handler's `DomainException` branch catches all subclasses and returns the embedded HTTP status automatically.

The throw-site mapping uses a generic "Invalid username or password" message for all login-context 401s to avoid user enumeration (do not distinguish "user not found" from "wrong password" at the API level).

## Tasks

### Task 1: Create typed domain exception classes in identity-service

**Files**:
- `anotame-api/backend/identity-service/src/main/java/com/anotame/identity/domain/exception/InvalidCredentialsException.java`
- `anotame-api/backend/identity-service/src/main/java/com/anotame/identity/domain/exception/UserAlreadyExistsException.java`
- `anotame-api/backend/identity-service/src/main/java/com/anotame/identity/domain/exception/ResourceNotFoundException.java`

**Action**: Create all three files in the `domain/exception/` directory (created by Plan 04-01 for `DomainException`).

```java
// InvalidCredentialsException.java
package com.anotame.identity.domain.exception;

import jakarta.ws.rs.core.Response;

public class InvalidCredentialsException extends DomainException {
    public InvalidCredentialsException() {
        super("Invalid username or password", Response.Status.UNAUTHORIZED);
    }
}
```

```java
// UserAlreadyExistsException.java
package com.anotame.identity.domain.exception;

import jakarta.ws.rs.core.Response;

public class UserAlreadyExistsException extends DomainException {
    public UserAlreadyExistsException(String username) {
        super("Username already taken: " + username, Response.Status.CONFLICT);
    }
}
```

```java
// ResourceNotFoundException.java
package com.anotame.identity.domain.exception;

import jakarta.ws.rs.core.Response;

public class ResourceNotFoundException extends DomainException {
    public ResourceNotFoundException(String resource) {
        super(resource + " not found", Response.Status.NOT_FOUND);
    }
}
```

**Verify**: `mvn compile -pl anotame-api/backend/identity-service -q` exits 0.

### Task 2: Replace `RuntimeException` throws in `AuthService.java`

**File**: `anotame-api/backend/identity-service/src/main/java/com/anotame/identity/application/service/AuthService.java`

**Action**: Edit. Add imports for the three new exception classes at the top of the file (after existing imports):

```java
import com.anotame.identity.domain.exception.InvalidCredentialsException;
import com.anotame.identity.domain.exception.ResourceNotFoundException;
import com.anotame.identity.domain.exception.UserAlreadyExistsException;
```

Replace each `throw new RuntimeException(...)` site as follows. The current file (post Phase 3 implementation) has these throw sites:

**`register()` method (line 29)**:
```java
// Before:
throw new RuntimeException("Username already taken");
// After:
throw new UserAlreadyExistsException(request.getUsername());
```

**`login()` method (line 57)** — "User not found" in login context:
```java
// Before:
.orElseThrow(() -> new RuntimeException("User not found"));
// After:
.orElseThrow(() -> new InvalidCredentialsException());
```
Note: Using `InvalidCredentialsException` here (not `ResourceNotFoundException`) prevents user enumeration. The caller cannot distinguish "no such user" from "wrong password".

**`login()` method (line 60)** — "Invalid credentials":
```java
// Before:
throw new RuntimeException("Invalid credentials");
// After:
throw new InvalidCredentialsException();
```

**`getUser()` method (line 88)** — "User not found" in a read context:
```java
// Before:
.orElseThrow(() -> new RuntimeException("User not found"));
// After:
.orElseThrow(() -> new ResourceNotFoundException("User"));
```

**`updateCredentials()` method (line 104)** — "User not found":
```java
// Before:
.orElseThrow(() -> new RuntimeException("User not found"));
// After:
.orElseThrow(() -> new ResourceNotFoundException("User"));
```

**`updateCredentials()` method (line 108)** — "Invalid current password":
```java
// Before:
throw new RuntimeException("Invalid current password");
// After:
throw new InvalidCredentialsException();
```
Note: `InvalidCredentialsException` is used here too — "Invalid username or password" is the generic message. This is intentional to avoid leaking which field failed.

**`updateCredentials()` method (line 115)** — "Username already taken":
```java
// Before:
throw new RuntimeException("Username already taken");
// After:
throw new UserAlreadyExistsException(request.getNewUsername());
```
