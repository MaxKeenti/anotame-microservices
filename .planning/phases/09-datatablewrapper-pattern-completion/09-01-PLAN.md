# Plan 09-01: Migrate Garments and Users Pages to DataTableWrapper

<objective>
Replace raw `Table.*` markup in the Garments and Users pages with the standardized `DataTableWrapper` component, gaining built-in sorting, filtering, and pagination for free. These two pages are the simplest targets — no external filter panels, straightforward column definitions.
</objective>

<must_haves>
- Garments page renders via DataTableWrapper with sorting on Name column
- Users page renders via DataTableWrapper with sorting on Name, Username columns
- Both pages retain Edit/Delete action buttons (admin-gated where applicable)
- Both pages retain their existing dialog components unchanged
- No raw `Table.Root` / `Table.Header` / `Table.Body` / `Table.Row` / `Table.Cell` markup remains
- `bun run build` exits 0
</must_haves>

---

<task id="09-01-01">
<title>Migrate Garments page to DataTableWrapper</title>

<read_first>
- anotame-web/src/routes/(app)/dashboard/catalog/garments/+page.svelte
- anotame-web/src/lib/components/ui/DataTableWrapper.svelte
- anotame-web/src/routes/(app)/dashboard/customers/+page.svelte (reference implementation)
</read_first>

<action>
1. Add imports at top of script block:
   ```
   import DataTableWrapper from '$lib/components/ui/DataTableWrapper.svelte';
   import type { ColumnDef } from '@tanstack/table-core';
   ```

2. Remove import of `* as Table` from `'$lib/components/ui/table'`.

3. Define column array after the `isAdmin` derived:
   ```typescript
   const columns: ColumnDef<any>[] = [
     { accessorKey: 'name', header: 'Nombre', enableSorting: true },
     { accessorFn: (row) => row.description || '-', id: 'description', header: 'Descripción', enableSorting: false },
     ...(isAdmin ? [{ id: 'actions', header: 'Acciones', enableSorting: false }] : []),
   ];
   ```
   NOTE: Since `isAdmin` is `$derived`, the columns array should be wrapped in `$derived` as well to react to role changes:
   ```typescript
   let columns = $derived<ColumnDef<any>[]>([
     { accessorKey: 'name', header: 'Nombre', enableSorting: true },
     { accessorFn: (row: any) => row.description || '-', id: 'description', header: 'Descripción', enableSorting: false },
     ...(isAdmin ? [{ id: 'actions', header: 'Acciones', enableSorting: false } as ColumnDef<any>] : []),
   ]);
   ```

4. Remove the `filteredGarments` derived and `searchQuery` state — DataTableWrapper has its own built-in search filter. Also remove the standalone `<Input>` search field from the template.

5. Replace the entire `<div class="bg-card ...">` table block (lines 91-147) with:
   ```svelte
   <div class="bg-card border border-border rounded-xl overflow-hidden shadow-sm p-4">
     <DataTableWrapper
       {columns}
       data={garments}
       {loading}
       emptyMessage="No se encontraron prendas."
       filterPlaceholder="Buscar prendas..."
     >
       {#snippet actionCell(row)}
         <div class="flex justify-end gap-2">
           <Button
             variant="outline"
             size="sm"
             class="h-10 px-4 touch-manipulation font-medium"
             onclick={() => handleEditClick(row.original)}
           >
             <Edit class="w-4 h-4 mr-2" />
             Editar
           </Button>
           <Button
             variant="outline"
             size="sm"
             class="h-10 px-4 text-destructive hover:text-destructive/90 touch-manipulation font-medium"
             onclick={() => handleDeleteClick(row.original)}
           >
             <Trash2 class="w-4 h-4 mr-2" />
             Eliminar
           </Button>
         </div>
       {/snippet}
     </DataTableWrapper>
   </div>
   ```

6. Keep the `GarmentDialog` component at the bottom unchanged.

7. Remove the standalone search `<div class="flex gap-2">` block (lines 83-89) since DataTableWrapper provides its own.
</action>
