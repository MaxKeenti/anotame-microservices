# Plan 10-01: Apply shadcn-svelte Preset and Selective Token Merge

<objective>
Run `bun x shadcn-svelte init --preset b4akO6QUQs` to apply the curated design preset, then selectively merge the preset's core tokens with the existing custom semantic tokens (warning/success/info/destructive variants) that were manually tuned for WCAG compliance.
</objective>

<must_haves>
The design token system is updated from the preset without losing custom semantic color tokens, and the build passes with zero errors.
</must_haves>

## Tasks

### Task 1: Backup current design system files

<read_first>
- anotame-web/src/routes/layout.css
- anotame-web/components.json
</read_first>

<action>
Create backups of the files that will be modified by the preset:

```bash
cp anotame-web/src/routes/layout.css anotame-web/src/routes/layout.css.backup
cp anotame-web/components.json anotame-web/components.json.backup
```

Also capture the current state of all shadcn components for diff comparison:
```bash
find anotame-web/src/lib/components/ui -name "*.svelte" -o -name "*.ts" | sort > /tmp/shadcn-components-before.txt
```
</action>

<acceptance_criteria>
- `anotame-web/src/routes/layout.css.backup` exists and matches the original file content
- `anotame-web/components.json.backup` exists and matches the original file content
- `/tmp/shadcn-components-before.txt` contains the list of 15+ component files
</acceptance_criteria>

### Task 2: Run shadcn-svelte init with preset

<read_first>
- anotame-web/components.json
- anotame-web/src/routes/layout.css
</read_first>

<action>
Run the shadcn-svelte init command with the user's specified preset from the `anotame-web/` directory:

```bash
cd anotame-web && bun x shadcn-svelte@latest init --preset b4akO6QUQs
```

Accept any prompts to overwrite existing files. The preset will:
1. Update `components.json` with the preset's configuration
2. Overwrite `src/routes/layout.css` with the preset's design tokens
3. Potentially regenerate shadcn components in `src/lib/components/ui/`

After running, capture the diff:
```bash
diff anotame-web/src/routes/layout.css.backup anotame-web/src/routes/layout.css || true
```
</action>

<acceptance_criteria>
- `bun x shadcn-svelte@latest init --preset b4akO6QUQs` completes without error
- `anotame-web/components.json` is updated (may contain new preset settings)
- `anotame-web/src/routes/layout.css` is modified with preset tokens
</acceptance_criteria>

### Task 3: Selective merge — restore custom semantic tokens

<read_first>
- anotame-web/src/routes/layout.css
- anotame-web/src/routes/layout.css.backup
</read_first>

<action>
Compare the preset-generated `layout.css` with the backup. The goal is to:

1. **KEEP from the preset:** All core design tokens that the preset changed:
   - `--background`, `--foreground`, `--card`, `--card-foreground`, `--popover`, `--popover-foreground`
   - `--primary`, `--primary-foreground`, `--secondary`, `--secondary-foreground`
   - `--muted`, `--muted-foreground`, `--accent`, `--accent-foreground`
   - `--destructive` (the base destructive color only)
   - `--border`, `--input`, `--ring`
   - `--chart-1` through `--chart-5`
   - `--radius`
   - `--sidebar-*` tokens
   - Any NEW tokens the preset introduces

2. **RESTORE from backup:** The custom semantic tokens (these are NOT standard shadcn tokens and won't be in the preset output):
   - `--success: oklch(0.696 0.17 162)` (light), `oklch(0.75 0.16 162)` (dark)
   - `--success-foreground: oklch(1 0 0)` (light), `oklch(0.145 0 0)` (dark)
   - `--success-muted: oklch(0.948 0.052 162)` (light), `oklch(0.25 0.06 162)` (dark)
   - `--success-text: oklch(0.37 0.12 162)` (light), `oklch(0.85 0.12 162)` (dark)
   - `--warning: oklch(0.75 0.15 75)` (light), `oklch(0.80 0.14 75)` (dark)
   - `--warning-foreground: oklch(1 0 0)` (light), `oklch(0.145 0 0)` (dark)
   - `--warning-muted: oklch(0.97 0.05 90)` (light), `oklch(0.26 0.05 90)` (dark)
   - `--warning-text: oklch(0.45 0.13 75)` (light), `oklch(0.88 0.10 75)` (dark)
   - `--info: oklch(0.65 0.15 250)` (light), `oklch(0.70 0.14 250)` (dark)
   - `--info-foreground: oklch(1 0 0)` (light), `oklch(0.145 0 0)` (dark)
   - `--info-muted: oklch(0.94 0.04 250)` (light), `oklch(0.24 0.05 250)` (dark)
   - `--info-text: oklch(0.4 0.12 250)` (light), `oklch(0.82 0.10 250)` (dark)
   - `--destructive-muted: oklch(0.95 0.04 25)` (light), `oklch(0.26 0.05 25)` (dark)
   - `--destructive-text: oklch(0.4 0.15 25)` (light), `oklch(0.85 0.12 25)` (dark)

3. **RESTORE from backup:** The `@theme inline` block must include the Tailwind v4 color mappings for ALL custom semantic tokens:
   - `--color-success`, `--color-success-foreground`, `--color-success-muted`, `--color-success-text`
   - `--color-warning`, `--color-warning-foreground`, `--color-warning-muted`, `--color-warning-text`
   - `--color-info`, `--color-info-foreground`, `--color-info-muted`, `--color-info-text`
   - `--color-destructive-muted`, `--color-destructive-text`

4. **PRESERVE:** The `@custom-variant` blocks (dark, data-open, data-closed, data-checked, data-unchecked, data-selected, data-disabled, data-active, data-horizontal, data-vertical) and the `@utility no-scrollbar` block. If the preset doesn't include them, restore from backup.

5. **PRESERVE:** `--font-sans: 'Inter Variable', sans-serif` in the `@theme inline` block.
</action>

<acceptance_criteria>
- `layout.css` `:root` block contains BOTH preset core tokens AND all 14 custom semantic tokens (success-*, warning-*, info-*, destructive-muted, destructive-text)
- `layout.css` `.dark` block contains matching dark-mode semantic tokens
- `layout.css` `@theme inline` contains `--color-success`, `--color-warning`, `--color-info`, `--color-destructive-muted`, `--color-destructive-text` mappings
- `layout.css` `@theme inline` contains `--font-sans: 'Inter Variable', sans-serif`
- `layout.css` contains all `@custom-variant` blocks (dark, data-open, data-closed, etc.)
- `layout.css` contains `@utility no-scrollbar`
- `grep -c "success\|warning\|info\|destructive-muted\|destructive-text" anotame-web/src/routes/layout.css` returns 28+ matches (14 tokens × 2 modes)
</acceptance_criteria>
