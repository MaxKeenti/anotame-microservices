# Plan 10-02: Regression Verification and Component Compatibility

<objective>
Verify that all pages and custom components work correctly after the preset application and component regeneration. Fix any breakages in DataTableWrapper, adaptive components, or page-level code caused by API changes in regenerated shadcn components.
</objective>

<must_haves>
All 7 data table pages, order wizard, KPI dashboard, and all dialogs render correctly in both light and dark modes with zero build errors.
</must_haves>

## Tasks

### Task 1: Build verification and import fix

<read_first>
- anotame-web/src/lib/components/ui/DataTableWrapper.svelte
- anotame-web/src/lib/components/ui/table/index.ts
- anotame-web/src/lib/components/ui/input/index.ts
- anotame-web/src/lib/components/ui/button/index.ts
</read_first>

<action>
Run the full build and fix any compilation errors:

```bash
cd anotame-web && bun run build
```

The most likely breakages after component regeneration are:
1. **Import path changes** — shadcn components may have changed their export structure (e.g., `Table.Root` → `TableRoot`, or named vs default exports)
2. **Prop API changes** — components may have renamed, added, or removed props
3. **Slot/snippet changes** — Svelte 5 snippet syntax may differ between versions

For each error:
1. Read the regenerated component's `index.ts` to understand the new export structure
2. Update the importing file to match the new API
3. Re-run `bun run build` until exit code 0

Critical files to check for import compatibility:
- `DataTableWrapper.svelte` — imports from `$lib/components/ui/table`, `$lib/components/ui/input`, `$lib/components/ui/button`
- `src/lib/components/ui/responsive/adaptive-confirm.svelte` — imports from `$lib/components/ui/alert-dialog`
- `src/lib/components/ui/responsive/adaptive-select.svelte` — imports from `$lib/components/ui/select`
- `src/lib/components/ui/responsive/adaptive-datetime-picker.svelte` — imports from `$lib/components/ui/popover`, `$lib/components/ui/calendar`
- All route pages in `src/routes/(app)/dashboard/**` — import from various UI components
</action>

<acceptance_criteria>
- `bun run build` exits with code 0
- `grep -r "from '\$lib/components/ui/" anotame-web/src/lib/components/ui/DataTableWrapper.svelte` shows valid imports that match regenerated component exports
- No TypeScript errors related to component props or missing exports
</acceptance_criteria>
