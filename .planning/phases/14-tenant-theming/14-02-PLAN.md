```typescript
import { PersistedState } from 'runed';

export type TenantTheme = {
  primaryColor: string | null;  // Hex format: "#FF6B6B", nullable
  fontFamily: 'Inter' | 'Outfit' | 'Merriweather' | null;  // Preset fonts or null
};

const DEFAULT_THEME: TenantTheme = {
  primaryColor: null,
  fontFamily: null,
};

// PersistedState automatically persists to localStorage, survives page reload
const _theme = new PersistedState<TenantTheme>('tenant_theme', DEFAULT_THEME);

export const tenantThemeStore = {
  get current(): TenantTheme {
    return _theme.current ?? DEFAULT_THEME;
  },

  set(updates: Partial<TenantTheme>) {
    _theme.current = { ...this.current, ...updates };
  },

  hasCustom(): boolean {
    return this.current.primaryColor !== null || this.current.fontFamily !== null;
  },

  reset() {
    _theme.current = DEFAULT_THEME;
  },
};
```

**Implementation notes:**

1. **Type TenantTheme:**
   - Matches backend EstablishmentJpa theme fields
   - fontFamily is an enum (string literal union) for type safety
   - Both fields nullable (null = use defaults)

2. **PersistedState pattern:**
   - Already used by paletteStore in the codebase
   - Auto-persists to localStorage key 'tenant_theme'
   - Survives page reload as fallback if server load fails
   - Include library: `import { PersistedState } from 'runed';`

3. **tenantThemeStore methods:**
   - `current`: Read-only getter (reactive)
   - `set()`: Update with partial object (e.g., only primaryColor)
   - `hasCustom()`: Utility to check if any customization exists
   - `reset()`: Clear all customization (debug/logout helper)

4. **No validation at store level:**
   - Validation happens in frontend form (sveltekit-superforms) and backend (optional, Phase 15)
   - Store just holds the values

**File location:** `anotame-web/src/lib/stores/tenant-theme.svelte.ts`

  <action>
Modify `anotame-web/src/routes/(app)/+layout.svelte` to initialize tenant theme store on mount and inject CSS variables reactively.

**Add to the script block:**

```typescript
<script lang="ts">
  import { tenantThemeStore } from '$lib/stores/tenant-theme.svelte';

  let { data, children } = $props();

  // Initialize store with server-loaded theme during hydration
  $effect.pre(() => {
    if (data.establishmentTheme) {
      tenantThemeStore.set(data.establishmentTheme);
    }
  });

  // Inject CSS variables whenever theme changes
  $effect(() => {
    const theme = tenantThemeStore.current;
    const root = document.documentElement;

    // Inject primary color if customized
    if (theme.primaryColor) {
      root.style.setProperty('--primary', theme.primaryColor);
    } else {
      root.style.removeProperty('--primary');  // Fallback to CSS default
    }

    // Inject font family if customized
    if (theme.fontFamily) {
      const fontMap = {
        'Inter': "'Inter Variable', sans-serif",
        'Outfit': "'Outfit Variable', sans-serif",
        'Merriweather': "'Merriweather Variable', serif",
      };
      root.style.setProperty('--font-sans', fontMap[theme.fontFamily]);
    } else {
      root.style.removeProperty('--font-sans');  // Fallback to CSS default
    }
  });
</script>
```

**Integration details:**

1. **Import:** Add `import { tenantThemeStore }` at top of script
2. **$effect.pre():** Runs before component renders
   - Initializes store with data.establishmentTheme from server load
   - Ensures theme is available before first render
   - Prevents FOUC

3. **$effect() for injection:** Runs after mount, re-runs when tenantThemeStore changes
   - Watches `tenantThemeStore.current` (reactive dependency)
   - Injects hex color directly to --primary variable
   - Maps fontFamily enum to full CSS font name (e.g., 'Outfit' → "'Outfit Variable', sans-serif")
   - Removes property if null (fallback to CSS defaults)

4. **Font mapping:**
   - 'Inter' → "'Inter Variable', sans-serif"
   - 'Outfit' → "'Outfit Variable', sans-serif"
   - 'Merriweather' → "'Merriweather Variable', serif"
   - Maps must match fonts @imported in layout.css

5. **No manual DOM attributes:**
   - Use `style.setProperty()` (safer than string concat)
   - Use `style.removeProperty()` (cleaner than null values)
   - Avoids CSS injection risks

**Placement in file:**
- Add at top of app layout (inside (app) route group)
- Before any child components render
- Alongside existing reactive effects

**Why $effect.pre() then $effect():**
- `$effect.pre()` initializes store (one-time, during hydration)
- `$effect()` injects CSS (reactive, updates when store changes, also fires on mount for FOUC prevention)
