# Quick Reference: totalDurationMin=0 Fix

## The Problem (1 Sentence)
When orders are updated, durationMin values are not copied to the database, causing totalDurationMin to be calculated as 0, breaking workload displays in dashboards.

---

## The Root Cause (Code Comparison)

### ✓ WORKS - createOrder() (line 88)
```java
service.setDurationMin(serviceDto.getDurationMin() != null ? serviceDto.getDurationMin() : 0);
```

### ✗ BROKEN - updateOrder() (line 217-232)
Missing the line above! Services created in update loop don't have durationMin set.

---

## The Fix (1 Line)

**File:** `SalesService.java`
**Location:** After line 226 in updateOrder() method
**Add this line:**
```java
service.setDurationMin(serviceDto.getDurationMin() != null ? serviceDto.getDurationMin() : 0);
```

That's it. Copy line 88 from createOrder() to line 227 in updateOrder().

---

## Data Flow (Why It Matters)

```
CREATE ORDER:
  Request (durationMin: 45)
    ↓
  SalesService.createOrder() sets durationMin ✓
    ↓
  OrderPersistenceAdapter saves to DB ✓
    ↓
  Database: duration_min = 45 ✓
  Response: totalDurationMin = 45 ✓

UPDATE ORDER:
  Request (durationMin: 45)
    ↓
  SalesService.updateOrder() does NOT set durationMin ✗
    ↓
  OrderPersistenceAdapter saves to DB (with NULL)
    ↓
  Database: duration_min = NULL ✗
  Response: totalDurationMin = 0 ✗ (sums null values)
```
