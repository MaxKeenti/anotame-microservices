# Milestone v1.3: Advanced Operations

**Status:** ✅ SHIPPED 2026-04-14
**Phases:** 15–17
**Total Plans:** 5
**Total Tasks:** ~13

## Overview

v1.3 completed the advanced order lifecycle: staff can now edit orders with role-based restrictions, deliver orders using a pickup code flow, perform bulk status changes and guarded bulk deletes, select a price list at order creation time (with auto-fill pricing), and configure the DataTable row count for constrained displays. All features are fully end-to-end verified against the live backend.

## Phases

### Phase 15: Order Lifecycle Improvements

**Goal:** Enable staff to edit existing orders and perform bulk actions on order lists.
**Depends on:** Phase 14
**Plans:** 3 plans

Plans:
- [x] 15-01: Backend — Flyway V3, pickup code, audit log, deliver endpoint, role-restricted update
- [x] 15-02: Frontend — Edit order wizard, pickup code display, status lock, i18n keys
- [x] 15-03: Frontend — Bulk actions, FloatingActionBar, pickup-code-dialog, operations READY tab

**Details:**
- Flyway V3 migration adds `pickup_code` column on `tco_order` and creates `tco_order_audit_log` table
- `SalesService.createOrder()` generates a 6-digit pickup code via ThreadLocalRandom
- `PATCH /orders/{id}/deliver` validates code with MessageDigest.isEqual() (constant-time)
- `updateOrder()` enforces EMPLOYEE role restrictions — notes/deadline/payment only; garment/service changes silently ignored
- Field-level audit log: one row per changed field per `updateOrder()` call (who, when, old, new)
- Edit route `/orders/[id]/edit` pre-populates wizard from `GET /orders/{id}` with `isEditing: true`
- Status lock: DELIVERED/CANCELLED orders show `role="alert"` banner; wizard CSS pointer-events-none
- `DataTableWrapper` extended with bulk mode: `bulkActions` prop, `bulkMode = $bindable()`, `onSelectionChange`, `__select__` checkbox column
- `FloatingActionBar`: role-aware status picker (ADMIN=5, EMPLOYEE=3), delete guard, cancel X
- `pickup-code-dialog`: 6-digit font-mono input, inline 400 error, `onDelivered` callback
- Operations page: "Listas para entrega" tab with READY orders; "Entregar pedido" opens dialog

**Verification:** PASSED 2026-04-13 — 9/9 must-haves verified

## Milestone Summary

**Key Decisions:**
- Phase 15: EMPLOYEE role check uses `EMPLOYEE` string literal (not OPERATOR) per JWT groups claim structure
- Phase 15: CSS pointer-events-none wrapper for locked wizard state (not per-step readonly props)
- Phase 15: `clearActiveDraft()` in edit mode (not `completeActiveDraft`) — in-memory clear only
- Phase 16: `priceListName` denormalized on order at creation (no runtime catalog fetch for display)
- Phase 16: No validation annotation on `priceListId` in CreateOrderRequest — stored as-is (cross-service validation out of scope)
- Phase 16: `OrderEntity` retains `@Getter @Setter` only (not `@Data`) to avoid Hibernate hashCode/equals recursion
- Phase 17: PersistedState without userId scoping — per-device preference (kiosk-style workstations)
- Phase 17: Init-time only — no live `$effect` tracking the store; table picks up preference on mount

**Issues Resolved:**
- Phase 15-03: `getRowSelectionRowModel` does not exist in @tanstack/table-core — removed, row selection native via `enableRowSelection: true`
- Phase 15-03: TypeScript type annotations in Svelte template expressions — extracted to named helper functions
- Phase 15-02: Pre-existing Snippet type import error in `+layout.svelte` (verbatimModuleSyntax)
- Phase 16 UAT: Price list not included in POST payload (missing field in payment-step)
- Phase 16 UAT: Service cards showed catalog price instead of price list price

**Issues Deferred:**

- Print Server Integration → moved to v1.5+ backlog
