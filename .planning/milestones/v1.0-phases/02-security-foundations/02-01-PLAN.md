# Phase 2 Plan 01: Untrack `.env` and harden `.gitignore`

**Phase:** 2
**Plan:** 01
**Goal:** Remove `.env` from git tracking and ensure all secrets files are permanently excluded from the repository.
**Req IDs:** SEC-01

---

## Overview

The `.env` file has been committed to git since at least commit `647d7b1` and contains plaintext local dev credentials. The `.gitignore` rule `.env*.local` does NOT cover the bare `.env` filename â€” it must be added explicitly. This plan stops the bleed: untrack `.env` without deleting it, strengthen `.gitignore`, and add the three missing placeholder vars (`PGADMIN_DEFAULT_PASSWORD`, `SMALLRYE_JWT_SIGN_KEY`, `MP_JWT_VERIFY_PUBLICKEY`) to both `.env` and `.env.example` so later plans can populate them.

### Task 1: Add `.env` exact-match rule to `.gitignore`

**File**: `.gitignore`

**Action**: Edit the "local env files" section (currently at line 448-449) to add a bare `.env` entry on its own line, separate from the glob pattern. The section must read:


### Task 2: Untrack `.env` from git index

**File**: (git index â€” no file modification)

**Action**: Run the following command from the project root. This removes `.env` from git's tracking index without deleting the local file:

```bash

```

Expected output: `rm '.env'`

After running, stage the `.gitignore` change and the `git rm --cached` removal together:

```bash
git add .gitignore
git status
```

`git status` must show:
- `.gitignore` as a modified staged file
- `.env` as "deleted" in the staged area (removed from tracking)
- `.env` appearing under "Untracked files" (still present on disk)

Do NOT commit yet â€” that happens after Task 3.

**Verify**: `git ls-files .env` returns empty (no output). The file still exists on disk: `test -f .env && echo "file exists"` must print "file exists".

### Task 3: Add missing env var placeholders to `.env` and `.env.example`

**File**: `.env`, `.env.example`

**Action**: Append the following block to `.env` (after the last existing line). These are concrete local dev values. The `PGADMIN_DEFAULT_PASSWORD` is already used in `docker-compose.yml` (plan 02-04 will wire it). The JWT vars will be populated in plan 02-02.

```
# --- PGADMIN ---
PGADMIN_DEFAULT_PASSWORD=password

# --- JWT KEYS (identity-service signs; all services verify) ---
# For local dev: populate after running plan 02-02 key generation.
# Value is the full PEM content including BEGIN/END headers.
# In .env files, multi-line values must be quoted with double quotes.
SMALLRYE_JWT_SIGN_KEY=
MP_JWT_VERIFY_PUBLICKEY=
```

Append the following block to `.env.example` (after the last existing line):

```
# --- PGADMIN ---
PGADMIN_DEFAULT_PASSWORD=<set-a-local-admin-password>

# --- JWT KEYS (identity-service signs; all services verify) ---
# Generate with: openssl genrsa -out privateKey.pem 2048 && openssl rsa -in privateKey.pem -pubout -out publicKey.pem
# Then export: export SMALLRYE_JWT_SIGN_KEY=$(cat privateKey.pem) and paste the full PEM value here.
# Multi-line PEM values must be wrapped in double quotes in .env files.
SMALLRYE_JWT_SIGN_KEY="-----BEGIN RSA PRIVATE KEY-----
<paste full key content here>
-----END RSA PRIVATE KEY-----"
MP_JWT_VERIFY_PUBLICKEY="-----BEGIN PUBLIC KEY-----
<paste full key content here>
-----END PUBLIC KEY-----"
```

**Verify**: `grep -n "PGADMIN_DEFAULT_PASSWORD" .env .env.example` â€” must show matches in both files. `grep -n "SMALLRYE_JWT_SIGN_KEY" .env .env.example` â€” must show matches in both files.
