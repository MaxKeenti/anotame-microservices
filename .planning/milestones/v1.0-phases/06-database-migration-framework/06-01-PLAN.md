  <action>
In each application.properties file, append a Flyway configuration block at the end of the file. The block is identical across services EXCEPT for the `quarkus.flyway.table` value.

For identity-service/src/main/resources/application.properties — append:
```properties

# Flyway — schema migration lifecycle
quarkus.flyway.migrate-at-start=true
quarkus.flyway.baseline-on-migrate=true
quarkus.flyway.baseline-version=1
# Per-service history table — prevents cross-service Flyway conflicts on shared anotame DB
quarkus.flyway.table=flyway_schema_history_identity

# Hibernate — disable auto-DDL in production; dev retains update for local convenience
%prod.quarkus.hibernate-orm.database.generation=none
```

For catalog-service — append (same block, different table name):
```properties

# Flyway — schema migration lifecycle
quarkus.flyway.migrate-at-start=true
quarkus.flyway.baseline-on-migrate=true
quarkus.flyway.baseline-version=1
quarkus.flyway.table=flyway_schema_history_catalog

%prod.quarkus.hibernate-orm.database.generation=none
```

For sales-service — append:
```properties

# Flyway — schema migration lifecycle
quarkus.flyway.migrate-at-start=true
quarkus.flyway.baseline-on-migrate=true
quarkus.flyway.baseline-version=1
quarkus.flyway.table=flyway_schema_history_sales

%prod.quarkus.hibernate-orm.database.generation=none
```

For operations-service — append:
```properties

# Flyway — schema migration lifecycle
quarkus.flyway.migrate-at-start=true
quarkus.flyway.baseline-on-migrate=true
quarkus.flyway.baseline-version=1
quarkus.flyway.table=flyway_schema_history_operations

%prod.quarkus.hibernate-orm.database.generation=none
```

IMPORTANT: Do NOT remove the existing `quarkus.hibernate-orm.database.generation=update` line. The bare property (no profile prefix) continues to apply in %dev, allowing local development without running migrations. The new `%prod.` line overrides it only in production (Railway). Two lines will coexist — this is correct Quarkus profile behavior.

IMPORTANT: `baseline-version=1` means Flyway stamps V1 as already applied on first contact with an existing database — V1__baseline.sql will never be executed against the live DB. Only V2 and later migrations run. This prevents the "Found non-empty schema but no history table" error.
